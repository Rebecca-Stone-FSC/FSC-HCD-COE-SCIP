Actions & Goals

What Users Are Doing
Actions + goals
😊Delight: Proactive

Reviewing prior-cycle rollover projects and cross-checking CAI data against expected baselines. Preparing station suspense schedules and confirming training compliance before granting access.

Identifying which stations need extra support or direct VISN-level project entry this cycle.

😕Friction: High Admin

CAI data imported as a locked snapshot. Granting and renewing station access. Reviewing baseline discrepancies visible at cycle open.

Correcting stations who have navigated into the prior-year cycle list.

😤Pain: Peak Friction

Monitoring station project submissions in real time. Reviewing descriptions, cost estimates, gap closure logic, project type selection, and checklist compliance.

Sending correction requests via message center. Entering projects directly for under-resourced stations.

😕Friction: Support Burden

Reviewing every project across all tabs. Validating description quality, cost estimate reasonableness, gap closure logic. Escalating unresolvable data issues to help desk.

Attempting to produce summary reports for leadership - mostly resorting to manual Excel extraction.

😕Friction: Pre-Submit

Final compliance check before approving and submitting the VISN project list to the program office. Trying to anticipate what central reviewers will flag.

No pre-flight compliance tool - relying entirely on experience and memory to catch errors.

😤Pain: Provisional Mode

Waiting on President's Budget release. Stations submit hedged, inflated, or duplicated projects as budget negotiations continue. Annual rework cycle begins.

SCIP data is intentionally provisional during this period - yet the system treats it as authoritative.

😕Friction: Facilitation

Central office returns reviewer comments. VISN routes feedback to stations, facilitates remediation, and tracks resubmission - some via message center, some via external spreadsheet.

No single communication standard. Stations uncertain where to receive feedback.

😤Pain: Manual Work

Generating reports for VISN leadership and program office. Built-in reporting tools are either undiscoverable, undertrained, or inadequate for required output formats.

Every VISN manager describes manually copying data from SCIP into Excel to create leadership summaries.

😐Neutral: Winding Down

Archiving completed cycle projects. Cleaning up station user access. Running space analysis tool comparisons for future cycle prep.

Lessons learned absorbed informally - never formally captured or surfaced for the next cycle.

Thoughts, Needs & Mental Models

Thoughts & Needs
Mental model · unmet needs

Mental model: capital planning is year-round work, not a seasonal sprint. Pre-cycle prep is as important as the cycle itself - experienced users feel this acutely.

Core need: know what's already in the station baseline before project entry begins, so CAI errors don't surface mid-cycle.

"I feel like it's year round. Maybe two months off from SCIP, in some sense."

Data visible in SCIP is already stale - CAI kept updating after the snapshot was locked. Experienced users know to call the help desk. Newer users don't, and the system doesn't tell them.

"There are certain errors based on data imported from CAI, where if changes have happened... that is not reflected in SCIP."

VISN managers are mentally pre-running the review process while stations are still entering data. They can predict the errors before seeing them.

Core need: in-tool guidance that prevents common errors upstream, so the VISN doesn't absorb the correction burden.

"However good or bad that estimate is, that's the number you get approved for."

VISN managers are doing station-level work. The system doesn't give station users enough guidance to succeed independently, so the correction burden flows upward.

This is the uncompensated support staff dynamic - a hidden cost with no visibility in any system metric.

"I want to run the same checklist the program office will run before I submit - catch everything they'll flag."

No pre-submission compliance view exists. Users rely entirely on experience and institutional memory. New users have no substitute.

"These submissions aren't real plans - they're negotiating positions. The number in SCIP right now is a placeholder until the budget drops."

⚠ Critical systemic finding: SCIP is used as a budget negotiation instrument during this window. Any analytics or AI tools built on in-cycle data would operate on intentionally provisional, inflated values - not planning data.

"Some reviewers use message center. Some send a spreadsheet. I don't always know where to look for feedback."

Fragmented review process creates confusion for both the stations receiving feedback and the VISN managers facilitating it.

"That whole reporting tab is meaningless to me right now. Training was offered once at 2.0 launch and I've lost it."

"I take all of that information out of SCIP but I don't actually export it from SCIP directly."

"Everything I learned this cycle lives in my head. If I leave, it goes with me."

No knowledge-capture mechanism exists. Institutional memory is concentrated in individuals. Each new person must rediscover everything from scratch.

Pain Points & Barriers

Pain Points
Barriers · blockers
⚠Baseline opacity: Cannot see what's in a station's space baseline without calling the help desk. Self-service is impossible.
⚠Space mapping tool removed: Tool mapping service types to SCIP space categories was removed from the UI; some users have a personal copy from years ago - most do not.
⚠No anomaly detection: Zero-workload data errors go undetected until an expert manually cross-references multiple external systems.
⚠CAI snapshot locked at cycle open: Baseline stales the moment the cycle opens. Mid-cycle CAI updates require manual help desk intervention - a bottleneck on every correction.
⚠Access expiration with no renewal path: Access expires every 45–90 days; users must be re-entered from scratch, not renewed. Incentivizes access gatekeeping.
⚠Cycle filter labeling confusion: "All budget year current cycle" routes users into prior-year cycle lists repeatedly.
🚨DEFECT - Checklist false-green: Business questions section shows complete even when empty. Allows incomplete submissions to advance past checklist undetected. Requires immediate investigation.
⚠Building gross conversion unknown: Estimator requires building gross SF; stations enter department gross. No in-tool prompt or reminder exists.
⚠Estimator lags industry 1–2 years: HVAC projects systematically underestimated. The approved SCIP figure becomes the approved project budget and cascades to the Congressional request.
⚠Checklist-to-tab mismatch: Checklist error items don't navigate to the correct form tab. Item order doesn't match tab order. Invisible to experts; a wall for infrequent users.
⚠Complex project type blocks: Conversion + lease combinations generate false gap errors. VHA/VBA shared-tenancy projects have no valid project type.
⚠Reporting tab effectively unusable for many: Capability exists but training was offered once at 2.0 launch and never repeated.
⚠No prioritization field: VISN managers cannot rank or prioritize projects within SCIP. Tracked in ad-hoc external spreadsheets.
⚠Review process fragmented: Some reviewers use message center, some use an external spreadsheet. No standard, no aggregate view.
⚠High-value features undiscovered: Station gap adjustment summary found only by accident after years of use.
⚠No pre-submission compliance view: No way to run the reviewer's checklist before submitting. Experienced users simulate it from memory.
⚠"VA Approved" status misleads: Interpreted as "funded" by station users. Means "approved for submission."
⚠"None" priority trap: Leaving priorities blank is an error; "None" must be explicitly selected. No in-form prompt.
⚠Budget cycle / SCIP cycle misalignment: President's Budget releases after SCIP submission deadline - annual rework is structurally baked into the process.
⚠Provisional data treated as authoritative: No mechanism to mark submissions as budget-dependent or provisional.
⚠No batch update tools: When budget releases, stations revise projects manually with no batch capability.
⚠Review tool fragmentation: External spreadsheet competes with message center. No aggregate view of comments across cycles, no pattern analysis.
⚠Status labels opaque: "In Remediation" label is unexplained; meaning inferred only through long experience.
⚠No standardized export: Every VISN manager builds a custom Excel workaround. No canned reports match what leadership reviews.
⚠Reporting regression from 1.0: The 2.0 tab is more powerful but far less discoverable. Capability without discoverability is invisible.
⚠Prioritization lives outside SCIP: No in-system field; tracked in ad-hoc per-person spreadsheets.
⚠Historical data loss (1.0→2.0 migration): One-week notice. Prior cycle data inaccessible. Trust significantly damaged.
⚠No knowledge capture: Lessons learned live in individuals, not the system. Tribal knowledge tax compounds with every staff transition.

Documented Workarounds - Shadow Systems & Compensating Behaviors

Workarounds
Shadow systems · compensating behaviors
⟳Calls help desk to confirm baseline composition each cycle - a self-service lookup that requires a ticket.
⟳Maintains personal desktop copy of the space mapping tool removed from SCIP years ago. Not all users have it.
⟳Manually cross-references VSSC workload, CAI gap data, and an external space analysis spreadsheet to detect data errors before they reach SCIP.
⟳Calls help desk to request manual SCIP database update for every known CAI discrepancy at cycle open.
⟳Maintains external Excel log of all station users and their expiration dates to proactively re-enter before access lapses.
⟳Sends screenshot-laden email reminders to stations every cycle showing which filter to select to reach the current cycle.
⟳Some VISN managers restrict access entirely and enter projects themselves to prevent errors from infrequent users.
⟳Proactively messages stations before the cost entry step to warn about building gross SF conversion - every cycle.
⟳Contacts help desk for manual override on every conversion + lease combination that generates a false gap error.
⟳Holds pre-cycle office hours specifically to pre-empt the same recurring errors every cycle - same questions, every year.
⟳Manually copies data from SCIP into Excel field by field to create leadership-facing project summaries.
⟳Tracks project prioritization in a personal Excel spreadsheet - an off-system record of what should be an in-SCIP field.
⟳Documents review findings in both message center AND external spreadsheet to cover inconsistency across reviewers.
⟳Mentally rehearses the central reviewer's checklist from memory before submitting - a cognitive substitute for a non-existent pre-flight compliance tool.
⟳Maintains external tracking spreadsheet for budget-pending project status while SCIP data remains provisional.
⟳Submits duplicate or inflated projects intentionally - knowing they'll be revised post-budget. A deliberate data quality compromise required by the process.
⟳Checks both message center and external review spreadsheet for feedback because no standard exists across central office reviewers.
⟳Builds Excel pivot tables from manual SCIP data extraction - rebuilt from scratch every cycle.
⟳Asks colleagues from other VISNs how to use reporting tools - informal peer support substituting for missing training.
⟳Uses message center comments as proxy for priority notation - not extractable, not auditable, not surfaceable.
⟳Relies on personal notes, institutional memory, and informal peer conversations to carry lessons from one cycle to the next.

Delight Signals - What Is Working

Delight Signals
Preserve + extend
★System supports year-round engagement for experienced users - project history, gap data, and rollover projects are durable across cycles.
★Custom views allow VISN managers to build filtered project lists tailored to their review workflow - flags, status, station, program type.
★Gap view is genuinely useful once found - directly shows which projects close which gaps.
★Station gap adjustment summary is described as "really handy" - when discovered.
★Message center: most-loved feature in the study. Threaded project communication with email notification and a direct deep-link to the project - eliminates inbox overhead for correction workflows.
"I love this message center. It allows me to talk to stations without sending a hundred emails. What I really love is it takes them directly to the project."
★Message center deep-link lands the user exactly at the project requiring correction - no re-navigation, no search, no wasted time.
★Cycle-over-cycle project continuity - rollover logic preserves institutional work history and eliminates re-entry of carried projects.
★Message center creates a communication audit trail - all project-level discussion captured in a single thread, visible to VISN and station.
★Reporting tab capabilities exist and are used successfully by trained VISNs - the data needed for leadership reports is available in the system.
★Historical project data accessible across cycles - supports longitudinal planning and gap trend analysis when not disrupted by migrations.

Design Opportunities

Opportunities

Pre-cycle data health dashboard: anomaly alerts, CAI deltas, baseline changes since last cycle

Restore space mapping tool to in-SCIP context at the point of space type selection

Baseline composition view for VISN users - self-service instead of help desk ticket

CAI–SCIP real-time sync, or delta view showing changes since cycle open - eliminates help desk bottleneck

Access renewal workflow - extend existing records without full re-entry

Rename cycle filters to explicit year labels: "SCIP 28 Cycle Projects"

In-tool guidance for manual CAI gap edit capability - capability exists, training does not

Fix checklist false-green defect - this is a bug, not a design improvement

Cost entry contextual reminder: "Have you converted department gross SF to building gross SF?"

Direct checklist-to-tab navigation links with ordering that matches tab sequence

Business rules support for conversion + lease companion project relationships

Conditional validation: energy priority checked → conservation priority field required

Default "None" in priorities, or prompt that blank is an error

Role-specific default view for VISN review: all projects, flag status, message count, submission status

Standardize review to message center - deprecate external spreadsheet tool

3–5 canned reports with one-click export matching what central leadership reviews

VISN-level priority/rank field - not shown in external submissions, exportable in reports

Surface station gap adjustment summary prominently in VISN review workflow

Pre-submission compliance report: run reviewer's checklist before submitting

Clarify "VA Approved" status: tooltip or inline explanation that it means approved for submission, not funded

Default "None" selection for priorities or in-form prompt that blank is an error

Budget status tags: allow submissions to be marked "provisional - pending President's Budget"

Batch update / rollover tools for post-budget project revision

Cycle status management: communicate to leadership that in-cycle data is provisional, not final

Caution flag for any analytics feature: in-cycle data reflects negotiating positions, not finalized plans

Mandate message center for all reviewer-to-submitter communication - eliminate tool split

Tooltip definitions for all status labels: "In Remediation," "VA Approved," etc.

Aggregate review comment view for VISN managers tracking multiple stations simultaneously

3–5 canned reports mirroring central leadership formats - one-click export to spreadsheet

Role-specific recorded training on reporting tools - not a one-time launch event

Audit existing feature utilization before building new features - undiscovery is the primary problem

VISN-level priority field with exportable report and leadership-facing sort order

Advance notice of future migrations - minimum 30 days, preferably 90

In-cycle notes that persist to the next cycle for the same users - begin knowledge capture

Evaluate access cleanup timing relative to late-cycle and post-cycle reporting needs
